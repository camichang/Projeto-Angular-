O que é Angular?

Antes de começarmos precisamos entender o que é Angular.

Angular é uma plataforma de aplicações web de código-fonte aberto e front-end baseado em TypeScript liderado pela Equipe Angular do Google e por uma comunidade de indivíduos e corporações.

Com a constante mudança e atualização já era esperado evoluir o AngularJs, sendo assim o Angular foi reescrito e adaptado, isto feito pela mesma equipe que o construiu.

Ele é baseado na linguagem TypeScript adicionando uma série de recursos a mais em especial relacionados a tipagem do JavaScript.

Conhecimentos básicos

Conforme dito anteriormente o Angular utiliza a linguagem Typescript que está relacionada com o JavaScript.

Para poder utilizar o Angular é necessário que você saiba HTML/CSS e claro JavaScript.

Arquitetura Angular

A arquitetura do Angular permite que possamos organizar a aplicação através de módulos, que fornecem um contexto para que os componentes sejam compilados.

Uma aplicação terá pelo menos um módulo raiz que vai inicializar a aplicação.

Também temos os componentes que são os responsáveis pela visualização. Os componentes utilizam serviços que fornecem funcionalidades específicas e que são indiretamente relacionadas a essas visualizações.

Serviços e componentes são simples classes com decoradores, que definem o tipo e fornecem metadados para informar o Angular como usá-los.

Módulos

Tem como objetivo declarar e agrupar tudo que criamos no Angular. Existem duas estrutura principais, que são: declarations e o providers.

Declarations é onde declaramos os itens que iremos utilizar nesse módulo, como por exemplo componentes e diretivas, já nos Providers informamos os serviços.

O ponto de entrada de uma aplicação Angular é o arquivo main.ts (gerado pelo CLI). Ela aponta para o módulo raiz, no caso abaixo é o AppModule.

Componentes

A maior parte do desenvolvimento quando se utiliza o framework Angular é feito nos componentes.

Cada componente define uma classe, que contém dados e lógicas do aplicativo e está sempre associada a um template HTML, onde são definidas as visualizações deste componente.

O decorator @Component() identifica a classe imediatamente como um componente e oferece o modelo e os metadados específicos dele.

Serviços e componentes são simples classes com decoradores, que definem o tipo e fornecem metadados para informar o Angular como usá-los.

Os metadados configuram, por exemplo, como o componente pode ser referenciado no HTML e também quais os serviços devem ser utilizados.

Diretivas

As diretivas são como marcadores no elemento DOM que comunicam ao Angular para incluir um comportamento específico.

O propósito da diretiva é fornecer meios para que possamos manipular o DOM ou estender as funcionalidades do elemento no qual a diretiva terá ação, como por exemplo alterar o seu conteúdo, estilo, visibilidade etc.., dependendo da necessidade da funcionalidade a ser desenvolvida.

Tipos de Diretivas:

  • ngStyle - serve para alterar em tempo real valores de propriedades de CSS dentro de um elemento HTML ou componente.
  • ngClass - serve para adicionar ou remover uma classe de algum elemento HTML ou componente que está sendo usado. A sintaxe é a mesma do ngStyle. Basta colocar a diretiva como atributo.
  • ngIf - O ngIf é uma diretiva estrutural que, a partir de uma condição booleana, pode ou não adicionar elementos a DOM, dinamizando ainda mais o site a interatividade do usuário.
  • ngSwitch - O ngSwitch também é uma diretiva estrutural e similar a estrutura switch das linguagens de programação.
  • ngFor - Com esta diretiva você pode utilizar laços de repetição para arrays que contém dados que serão apresentados na tela do usuário.

Componentes Auxiliares podem facilitar ainda mais a utilização dessas diretivas. Esses componentes são:

  • ng-template - Esse componente guarda dentro dela elementos HTML ou componentes que podem ou não serem renderizados. Normalmente é utilizado para mostrar outros elementos quando alguma condição feita com o ngIf é falsa, sendo um componente alternativo a ser renderizado na tela
  • ng-container - Esse componente tem uma proposta similar ao ng-template. No entanto, ele é usado normalmente para renderização em diretivas ngFor que possuem uma condição com ngIf.
  • ng-content - Com o ng-content é possível fazer com que elementos ou componentes sejam adicionados como filhos de outro componente, sendo mostrados quando o componente for renderizado em tela.
